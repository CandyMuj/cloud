二. Git和GitHub

* 什么是Git

Git 是目前世界上最先进的开源的分布式版本控制系统（没有之一）。

版本控制是指对软件开发过程中各种程序代码、配置文件及说明文档等文件变更的管理，是软件配置管理的核心思想之一。

 

* 什么是GitHub

GitHub是一个面向开源及私有软件项目的托管平台，因为只支持git 作为唯一的版本库格式进行托管，故名GitHub。

简单来说GitHub是基于Git的一个线上仓库，当然他除了Git代码仓库托管及基本的 Web管理界面以外，还提供了订阅、讨论组、文本渲染、在线文件编辑器、协作图谱（报表）、代码片段分享（Gist）等功能。

上方所说的同样是Git和GitHub之间的关系

 

三. 安装Git

上方说到，GitHub实质是基于Git的一个线上仓库，所以我们需要先使用Git的本地仓库，然后再将我们的本地仓库的内容或变更推送到线上。这也是使用GitHub的最基本的流程。

五. 配置GitHub SSH key（关联线上仓库GitHub的线上账号）

* 准备和说明

  1. 首先你需要一个GitHub的账号（线上账号）

    至于如何注册，我就不赘述了，直接“Sign up”注册一个就行了， 只需要输入用户名、邮箱、密码即可，很简单的，注册时后方的会有一些初始设置，看着选，看着填吧，对于你后续的使用没有什么影响

  2. 为当前设备添加SSHkey

    • SSH是一种网络协议，用于计算机之间的加密登录，使用这种方式登录是安全的
    • 如果不进行配置，那么你每次推送代码到线上，每次都可能提示你输入账号密码进行登录；但一般情况下，只是首次需要你进行登录，且本次登录以后clone此账号下的其他仓库也将获得权限。
    • 使用SSH的配置方式可以一劳永逸，一次配置成功以后，在本设备任意软件内使用Git的ssh链接方式操作都无需登录，将自动使用SSHkey作为认证方式登录，因为使用https的方式是由当前使用的软件保存你的登录凭证，是以软件为单位认证，而key是以设备为单位认证，将更加方便。
    • 一个设备有且仅需设置一个key即可，除非你把本机已有的key删除了（包括GitHub上的删除和本地的删除），那么才可以再次生成key
    • 当然你可以可以选择不配置此key，那么第五这个步骤可直接不看了，你可以在弹出登录框或提示登录的时候使用用户名密码的方式进行验证，但此方法可能会出现频繁登录的情况，并且凡使用git的软件都需要单独输入账号密码进行认证，这将会很繁琐

 

* 配置SSH key步骤

  1. 登录你的GitHub账号 进入 https://github.com/login

  2. 进入 https://github.com/settings/keys

  3. 如果在页面中显示的有key（下图红框位置，这个图就代表没有key），请确认这些key是否正在使用，删除一些没必要存在的key（如果有，那么每一个key后方都有一个delete按钮，点击即可删除），若没有请看下一步

  4. 在右键菜单中打开 Git Bash

  5. 复制并运行 rm -rf ~/.ssh/*删除本机已有的所有sshkey，如果有的话他会删除，没有的话他不会做任何更改

    注意：此命令一定要复制粘贴，且完全一致，如果有任何不同，哪怕是多了个空格，可能都会导致系统崩溃，只能重装系统了

  6. 复制并运行ssh-keygen -t rsa -b 4096 -C "your_email@example.com"，将your_email@example.com替换为你的邮箱，此邮箱可随意填写你的真实邮箱，建议填写为你GitHub账号的邮箱

  7. 在接下来让你输入内容的时候，直接回车，不要输入任何内容，按三次回车，随后你会看到类似下图的界面

    当你使用命令 ll ~/.ssh时会发现生成了如图红框内的两个文件

  8. 再次进入 https://github.com/settings/keys，并点击右上角的New SSH key新建一个sshkey（SSH keys的右边那个按钮）

  9. 继续执行命令cat ~/.ssh/id_rsa.pub查看生成的秘钥，你会看到如图所示的一串字符，完整复制这一串字符，下一步会用到

  10. 按图中所示填入相应内容点击添加即可

    如果提示输入密码进行验证，那么输入密码进行确认

  11. 回到Git Bash，执行ssh -T git@github.com，测试你的SSH 连接

    你可能会看到类似下方的提示

    xxxxxxxxxx
    The authenticity of host 'github.com (IP ADDRESS)' can't be established.
    RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
    Are you sure you want to continue connecting (yes/no)?

    或者是下方这样的

    xxxxxxxxxx
    The authenticity of host 'github.com (IP ADDRESS)' can't be established.
    RSA key fingerprint is SHA256:nThbg6kXUpJWGl7E1IGOCspRomTxdCARLviKw6E5SY8.
    Are you sure you want to continue connecting (yes/no)?

    那么直接键入yes即可，如果没有出现提示那么就跳过此步，可能直接成功了，也可能是直接失败了，下一步说明什么表示成功，什么表示失败

  12. 如果看到Error: Permission denied (publickey)就说明失败了，首先你可以多次尝试测试连接（上一步，第11步，因为有时可能是网络原因造成的失败），如果仍然不行，那么请回到第一步重新开始，如果你看到类似Hi username! You've successfully authenticated, but GitHub does not provide shell access.的提示（类似下图）

     

    那么就说明你成功了，恭喜你！

    好了，添加了一SSH key，接下来就会用到它。

     

* 总结和说明

  • 一台电脑只需要一个 SSH key

  • 一个 SSH key 可以访问你的所有仓库，即使你有 1000000 个仓库，都没问题

  • 如果你新买了电脑，就在新电脑上重新生成一个 SSH key，把这个 key 也上传到 GitHub，它可以和之前的 key 共存在 GitHub 上

  • 如果你把 key 从电脑上删除了，重新生成一个 key 即可，替换之前的 key
